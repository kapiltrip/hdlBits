## What the question is asking

The assignment is `y[1]=A`, `y[2]=B`, ..., `y[6]=F`. A one-hot next-state bit is high when the FSM takes **any enabled arrow entering that destination state**. This incoming-arrow method directly produces a sum-of-products equation and still behaves correctly when the testbench deliberately supplies a non-one-hot input vector.

### Deriving `Y2`

`Y2` means “the next state is B.” The only incoming transition is `A --w=0--> B`, so:

```verilog
assign Y2 = y[1] & ~w;
```

### Deriving `Y4`

`Y4` means “the next state is D.” Four arrows enter D under `w=1`: from B, C, E, and F. Therefore:

```verilog
assign Y4 = w & (y[2] | y[3] | y[5] | y[6]);
```

State bit `y[4]` does not appear because no transition from D returns to D. The resulting unused-input synthesis warning is therefore expected and was visible beside the successful HDLBits result.

## Saved Verilog solution

```verilog
module top_module (
    input [6:1] y,
    input w,
    output Y2,
    output Y4
);
    assign Y2 = y[1] & ~w;
    assign Y4 = w & (y[2] | y[3] | y[5] | y[6]);
endmodule
```

## Points to remember

- Next-state bit = OR of all enabled incoming transitions.
- Inspect arrows entering B and D, not arrows leaving them.
- Each incoming arrow becomes one AND term.
- Do not use a priority `case`; the testbench may assert several `y` bits.
- An absent current-state bit can be a correct consequence of the state graph.
