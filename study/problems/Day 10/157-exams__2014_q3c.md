## What the question is asking

The input `y[2:0]` is the **present state**. The table supplies next state `Y[2:0]`, but the module asks only for its least-significant bit `Y0` and the present-state output `z`. Therefore `Y0` must be derived from bit 0 of each selected next-state entry; it is not simply `y[0]`.

That observation is correct. The remaining step is to express that next-state bit as a Boolean function of the **current** `y` and `x` values.

## Handwritten derivation

![Handwritten Karnaugh-map derivation for Q3c Y0](../../images/Day%2010/157-exams__2014_q3c-handwritten-derivation.jpeg)

Splitting the table by input reduces the derivation to two present-state conditions:

| `x` | Legal present states where `Y0=1` | Simplified condition |
|---:|---|---|
| 0 | `001`, `011`, `100` | `y[0] \| y[2]` |
| 1 | `000`, `010` | `~y[0] & ~y[2]` |

Combining the two columns gives:

```verilog
assign Y0 = (~x & (y[0] | y[2])) |
            ( x & ~y[0] & ~y[2]);
```

For legal states this is also `x ^ (y[0] | y[2])`, but the expanded form is easier to audit against the table. Output `z` is independent of `x` because it is a Moore output and is high only in present states `011` and `100`.

## Saved Verilog solution

```verilog
module top_module (
    input clk,
    input [2:0] y,
    input x,
    output Y0,
    output z
);
    assign Y0 = (~x & (y[0] | y[2])) |
                ( x & ~y[0] & ~y[2]);
    assign z = (y == 3'b011) || (y == 3'b100);
endmodule
```

## Points to remember

- Lowercase `y` is present state; uppercase `Y` is next state.
- Select the requested next-state bit before simplifying.
- Derive Moore output `z` separately from the present-state column.
- Exact state comparisons avoid assigning accidental meaning to unused encodings.
- `clk` is intentionally unused because this exercise asks only for logic functions.
